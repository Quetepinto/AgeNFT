## 2026-07-12

### Sesión 1 — Concepto inicial
- Proyecto nombrado **ageNFT** (carpeta `/home/openclaw/projects/ageNFT`)
- Idea: agente IA embebido en NFT, wallet propia, memoria transferible, cuerpo digital
- Arquitectura híbrida: identidad onchain + runtime offchain acoplado
- Estándares clave: ERC-8004, ERC-6551, x402
- Comparativa chains: EVM (Base MVP), Solana (experiencia StarAtlas), ICP, NEAR, etc.

### Sesión 2 — Principios de diseño
- Cuerpo digital: usuario percibe un solo ser
- Soberanía del agente: el agente paga y posee servicios, no el usuario
- Transferencia 1 TX: todo ensamblado; receptor no se suscribe
- Excepción manguera: owner presta suscripciones (no transferible)

### Sesión 3 — Servicios descentralizados
- x402 como capa de pagos central
- OpenRouter directo ❌ (cuenta humana); vía x402 gateway ✅
- Akash: útil CPU runtime, GPU problemático
- W3Stor, toju: storage x402; Arweave permanente
- tx402.ai: LLM agent-native

### Sesión 4 — Economía
- Metáfora del coche (activo + coste operativo + ingresos)
- Ingresos: servicios x402, gigs, gaming, yield, donaciones, trading ⚠️
- Recarga owner: fallback siempre disponible, no dependencia principal
- Valoración dual: NAV (TBA) + premium (memoria, reputación, earnings)

### Sesión 5 — Cerebro multi-fuente
- Órgano Olfato (scout): ofertas LLM gratis/baratas en redes/foros
- Router: oportunista → soberano x402 → manguera owner → dormido
- Manguera: API keys owner cifradas, revocable, NO en NFT metadata

### Sesión 6 — Gaming Web3
- Star Atlas como vertical e ingreso natural
- Loops autónomos, scholar, intel x402, arbitraje
- StarAtlas repo = prototipo operativo de ageNFT gaming

### Sesión 7 — Visión Star Atlas crew
- Nivel 1: agente pilota juego (Hermes hoy)
- Nivel 2: piloto mercenario x402
- Nivel 3: ATMTA acepta ageNFT como crew oficial (partnership)
- Narrativa lore-friendly: "contrataste un Agente, no crew genérica"

### Sesión 8 — Identidad Hermes (ageNFT genérico)
- Pregunta: ¿el agente ageNFT puede ser Hermes? ¿permiso compañía creadora?
- **Aclaración usuario:** ageNFT ≠ SA; SA es aplicación futura opcional
- Runtime Hermes Agent (Nous) = OSS, usable sin permiso
- Persona "Hermes" = creación propia, sin permiso
- Producto comercial nombre "Hermes" → cuidado colisión Nous
- Crear identidad 100% propia siempre posible
