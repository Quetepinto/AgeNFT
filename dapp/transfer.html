Qué viaja al transferir el NFT

Un ageNFT es un cuerpo digital transferible. Una sola transacción onchain y el comprador hereda el paquete completo.

Viaja con el token ✅

  • Identidad — NFT + nombre + metadata onchain
  • Tesoro (TBA) — USDC, ETH y otros activos en la cuenta del token
  • Manifiesto — cerebro, memoria, chat, presupuesto (ageNFT/v1)
  • Memoria operativa — cápsulas locales / punteros offchain ligados al agente
  • Reflejos — límites de gasto definidos en manifiesto

No viaja automáticamente ⚠️

  • Runtime / servidor — quien ejecuta el agente (Hermes, VPS) es infra del operador
  • Claves del owner anterior — el nuevo dueño usa su wallet; la TBA paga con su saldo
  • Telegram / gateways — hay que reconfigurar credenciales con el nuevo operador
  • Unit-1 Sepolia — token lab legacy; no se migra a mainnet

Checklist post-transfer (MVP)

  1. Verificar owner del NFT en BaseScan
  2. Comprobar saldo USDC/ETH en la TBA
  3. Actualizar runtime con manifiesto unit-mainnet.json
  4. Probar un turno: npm run once:pay (payer TBA)
  5. Reconfigurar Telegram/Hermes si aplica
